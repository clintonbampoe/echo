Echo is a modular web application for church finance, membership, attendance, and reporting.
This repository is the central hub for both source code and documentation. It contains planning notes, design references, and implementation work, serving as the single source of truth for contributors throughout development.

---

## What this repository contains

- **Documentation**: Scope, requirements, design references, and implementation notes.  
- **Source code**: Application modules and tests (added as development progresses).  
- **Legacy assets**: Existing Excel sheets, CSV exports, and migration files.  
- **Media**: Design diagrams, mockups, and other visual assets.  
- **Configuration**: CI/CD pipelines, templates, and automation scripts.

## Project principles

- **Docs first** Decisions, requirements, and designs are recorded in documentation before implementation.  
- **Modular design** Features are split into independent modules to enable safe iteration and reuse.  
- **Traceability** Every change should be traceable to a requirement, issue, or design note.  
- **Documentation is code** Treat docs and process artifacts as first-class deliverables that must be kept current.
