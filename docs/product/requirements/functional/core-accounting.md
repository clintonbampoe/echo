# Core Accounting Requirements



## Income Management

- **CA-IM001**: The system must allow entry of income records with fields for source, category, amount, and date.

- **CA-IM002**: The system must support multiple income sources (services, events, contributions) and allow categorization.

- **CA-IM003**: The system must validate income entries to prevent invalid or negative amounts.

- **CA-IM004**: The system must automatically update balances when new income is recorded.

- **CA-IM005**: The system must support recurring income entries with configurable frequency (weekly, monthly, yearly).

- **CA-IM006**: The system must provide search and filter options for income records by category, source, or date.

## Expenditure Management

- **CA-EM001**: The system must allow entry of expenditure records with fields for category, purpose, amount, and date.

- **CA-EM002**: The system must support linking expenditures to specific projects or designated purposes.

- **CA-EM003**: The system must validate expenditure entries to prevent invalid or negative amounts.

- **CA-EM004**: The system must automatically update balances when new expenditure is recorded.

- **CA-EM005**: The system must provide search and filter options for expenditure records by category, project, or date.

- **CA-EM006**: The system must allow categorization of expenditures into operational, capital, welfare, or event-related categories.

## Balances

- **CA-BL001**: The system must provide an overview of balances across all accounts (cash, bank, designated funds).

- **CA-BL002**: The system must automatically update balances whenever income or expenditure records are added, edited, or removed.

- **CA-BL003**: The system must allow balances to be tracked periodically (daily, weekly, monthly, yearly).

- **CA-BL004**: The system must enable authorized users to make manual adjustments (e.g., bank charges, corrections) with audit logs.

- **CA-BL005**: The system must maintain historical balance records to support reviews and audits.

- **CA-BL006**: The system must flag discrepancies between expected balances (based on transactions) and manually entered closing balances.
